Zdarzenie Początkowe (Start Event): • Nazwa: Klient zgłasza problem techniczny • Typ: Prosty (bez specjalnych triggerów) • Reprezentacja: Koło z pojedynczą cienką linią. 2. Czynność (Task): • Nazwa: Agent obsługi klienta rejestruje zgłoszenie • Typ: Standardowa czynność (Abstract Task) • Reprezentacja: Prostokąt z zaokrąglonymi rogami. • Przepływ wychodzący (Sequence Flow): Do następnej czynności. 3. Czynność (Task): • Nazwa: System automatycznie przeszukuje bazę wiedzy • Typ: Czynność usługowa (Service Task)- sugeruje automatyczne działanie systemu. • Reprezentacja: Prostokąt z zaokrąglonymi rogami i ikoną zębatki (lub innej ikony symbolizującej usługę). • Przepływ wychodzący (Sequence Flow): Do następnej czynności. 4. Czynność (Task): • Nazwa: Agent weryfikuje proponowane rozwiązania • Typ: Czynność użytkownika (User Task)- sugeruje interakcję z człowiekiem. • Reprezentacja: Prostokąt z zaokrąglonymi rogami i ikoną ludzika. • Przepływ wychodzący (Sequence Flow): Do bramki rozłącznej. 5. Zadanie 1: Proces podejmowania decyzji kredytowej z wykorzystaniem reguł biznesowych. • Rozpoczęcie: Klient składa wniosek o kredyt (Zdarzenie Początkowe). • Czynności:– System zbiera dane klienta (Zasób: Dane Klienta).– System uruchamia silnik reguł biznesowych (Zasób: Silnik Reguł).– Silnik reguł ocenia wniosek na podstawie zdefiniowanych reguł (Zasób: Reguły Biznesowe- np. dotyczące historii kredytowej, dochodów).– Warunek:Jeśli wniosek spełnia wszystkie kryteria, system automatycznie akceptuje wniosek.– Warunek: Jeśli wniosek nie spełnia niektórych kryteriów, system może go odrzucić lub skierować do analityka kredytowego.– Analityk kredytowy ręcznie analizuje wniosek i dodatkowe dokumenty.– Analityk podejmuje decyzję (akceptacja lub odrzucenie).– System informuje klienta o decyzji. • Zasoby: Klient, System Bankowy, Silnik Reguł Biznesowych, Reguły Biznesowe, Dane Klienta, Analityk Kredytowy. • Czas: Czas złożenia wniosku, czas oceny przez silnik reguł, czas analizy analityka, czas odpowiedzi dla klienta. • Kolejność: Złożenie wniosku → Zbieranie Danych → Uruchomienie Silnika Reguł → Ocena Reguł → (Automatyczna Akceptacja LUB Odrzucenie LUB Analiza • Analityka → Decyzja Analityka) → Informowanie Klienta. • Zakończenie: Wniosek kredytowy został rozpatrzony (Zdarzenie Końcowe). Zadanie 3: Proces rekomendacji produktu w sklepie internetowym z wykorzystaniem profilowania klienta i reguł asocjacyjnych. • Rozpoczęcie: Klient przegląda produkty w sklepie internetowym (Zdarzenie Początkowe). • Czynności:– Systemśledzi aktywność klienta (przeglądane produkty, dodane do koszyka- Zasób: Dane Aktywności Klienta).– System analizuje profil klienta (historia zakupów, preferencje- Zasób: Profil Klienta).– System wykorzystuje reguły asocjacyjne (Zasób: Reguły Asocjacyjne- np. ”klienci kupujący X kupują także Y”).– Warunek: Jeśli system zidentyfikuje produkty powiązane z zainteresowaniami klienta, wyświetla rekomendacje.– Klient może zaakceptować rekomendacje i dodać produkty do koszyka. • Zasoby: Klient, System Sklepu Internetowego, Profil Klienta, Dane Aktywności Klienta, Reguły Asocjacyjne. • Czas: Czas przeglądania produktów, czas analizy przez system, czas wyświetlania rekomendacji. • Kolejność: Przeglądanie Produktów → Śledzenie Aktywności → Analiza Profilu → Wykorzystanie Reguł Asocjacyjnych → (Wyświetlenie Rekomendacji) → Akceptacja Rekomendacji (opcjonalne). • Zakończenie: Sesja przeglądania sklepu (Zdarzenie Końcowe) lub dokonanie zakupu. 6
